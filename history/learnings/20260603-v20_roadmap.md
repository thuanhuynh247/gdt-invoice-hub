## 1. Architectural Lessons & Takeaways

### A. Database-Backed P2P Mailroom for Agent Swarms
- **Lesson:** In a multi-agent orchestration setup, synchronous HTTP calls or complex in-memory message queues between agents are fragile and hard to debug. Implementing a database-backed inbox/outbox table (`AgentMessage`) decouples execution, permits offline message auditing, and enables resilient message retrieval.
- **Pattern:** Using simple SQLite JSON fields for payloads lets different agents communicate dynamically without needing migrations for every new agent type or message schema.

### B. High-Precision Normalization of Heterogeneous Bank Feeds
- **Lesson:** Bank transaction statements from different commercial banks (VCB, Techcombank, etc.) vary widely. Normalizing them into a single `BankLedger` schema first, then running a structured matcher with weighted scoring rules (MST matching, text similarity of payment references, and numeric value tolerance) provides an extremely high-accuracy matching process.
- **Pattern:** Explicitly warning when invoice totals exceed 20M VND without a matching bank transaction (non-cash payments rule under Vietnamese VAT law) directly helps users avoid compliance penalties.

## 2. Technical Decisions & Refinements

### A. Seasonality-Aware ML Tax Projections
- **Tet Holiday Seasonality:** Incorporating historical seasonality is critical for Vietnamese financial applications. Projections that ignore the Tet holiday drop produce highly inaccurate tax liability estimates. Seasonality-aware linear regression combined with moving averages produces much more reliable predictions.
- **Confidence Intervals:** Displaying 80% and 95% confidence bands visually communicates prediction variance, helping financial planners prepare for best and worst-case cash flow scenarios.

### B. Scenario Simulation Sandbox
- **Decree 132 Compliance:** Formulating CIT simulation to respect the 30% EBITDA interest deductibility cap ensures that simulated scenarios are legally valid. Non-deductible interest must be calculated as: `EBITDA = net_profit + depreciation + related_party_interest_expense`, adding back the excess interest to taxable income.

## 3. Critical Patterns Promoted

1. **Standardized ISO 20022/Bank Normalization Pattern:**
   - Map all bank statements to a unified schema (`BankLedger`) containing transaction date, amount, accounts, MST, and memo before executing match heuristics.
2. **EBITDA Compliance Formula Execution:**
   - Standardize calculations of EBITDA in python matching the strict definitions of Vietnamese tax decree 132 (incorporating related party interest add-backs) to ensure compliance.
