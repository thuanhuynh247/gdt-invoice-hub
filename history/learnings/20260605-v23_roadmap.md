## 1. Architectural Lessons & Takeaways

### A. Exporter VAT Refund Validation (Circular 80/2021/TT-BTC)
- **Lesson:** Exporters face stringent tax audits when claiming VAT refunds. Verifying customs declarations against domestic invoices based on description, quantity, and total currency, combined with enforcing the non-cash payment rule (> 20M VND must match bank transactions), greatly reduces audit exposure.
- **Pattern:** Compile Form 01/ĐNHT XML packets containing detailed invoice lists, supplier tax statuses, and customs tracking IDs to prepare audit-ready filings.

### B. Secure Versioned REST Gateway
- **Lesson:** Secure integrations with enterprise ERPs must prevent replay attacks and message tampering without introducing session states.
- **Pattern:** Authorize incoming REST requests by calculating an HMAC-SHA256 signature from `timestamp + "." + query_string_or_body` using a shared client secret. Verify the signature against the `X-GDT-Signature` header and reject any request with an `X-GDT-Timestamp` older than 300 seconds.

## 2. Technical Decisions & Refinements

### A. Webhook Hub Retry Queue
- **Stateful Retries:** Dispatched webhook events may fail due to recipient system offline periods.
- **Backoff Algorithm:** Execute an exponential retry backoff schedule (delay = $2^{retry} \times 10$ seconds) capped at 3 retries. If the final retry fails, flag the subscription as degraded to let users review the logs.

### B. Offline RAG Fallbacks
- **Resilient AI Auditing:** Local LLM instances (like Ollama) are prone to connection timeouts or setup failures under offline environments.
- **Regex Search Fallback:** When Ollama is unreachable, fall back to structural keyword searching (e.g. matching "Đơn giá", "Thời điểm xuất", "Decree 123") in local JSON files to extract relevant regulations, preventing system crashes.

## 3. Critical Patterns Promoted

1. **HMAC Request Signing Gate:**
   - Always validate the signature payload using client keys, and immediately reject any timestamp that deviates by more than 300 seconds from the server's time to thwart replay attacks.
2. **Local AI/RAG Resiliency Pattern:**
   - Design RAG interfaces with a fallback mock or regex keyword lookup so that query endpoints return sensible local information when the Ollama service is offline.
3. **Circular 80 Non-Cash Payment Constraint:**
   - Enforce bank ledger transaction reconciliation for all input invoices over 20M VND, marking them as non-eligible for refund claims if cash payment is flagged.
